## Breaking Changes

### Unleash Proxy instead of Server API

v1 used the Unleash **Server API** with a backend token and the `unleash-client` Node.js SDK. v2 uses the **Frontend API** via your Unleash Proxy with a frontend token.

```ts [nuxt.config.ts]
// v1
unleash: {
  url: 'https://unleash.example.com/api',  // Server API
  token: 'server-api-token',               // Backend token
}

// v2
unleash: {
  url: 'https://your-proxy.example.com/api/frontend',  // Frontend API
  token: 'frontend-api-token',                          // Frontend token
  storage: 'memory',                                    // New: storage backend
}
```

### Removed: `useUnleashContext()`

The `useUnleashContext()` composable has been removed. With the Frontend API and global flags, per-user context changes on the client are not needed.

### Removed: `disableMetrics`

The `disableMetrics` option has been removed. The Frontend API does not handle metrics.

### Changed: API endpoint

The internal API endpoint changed from `/api/_unleash/evaluate` to `/api/_unleash/flags`.

### New: `storage` option

v2 requires choosing a storage backend. The default is `memory`, which works for development and simple Node.js deployments. For edge deployments, use `nuxthub` or `nitro`.

## Migration Steps

::steps
### Update your config

Replace the Unleash Server API URL with your Proxy Frontend API URL. Change the token to a frontend token.

### Choose a storage backend

Add `storage: 'nuxthub'` or `storage: 'nitro'` if deploying to edge. See [Storage](/docs/storage/overview).

### Remove `useUnleashContext()` calls

Search your codebase for `useUnleashContext` and remove any usage. Flags are now global and don't change per user context.

### Update API endpoint references

If you called `/api/_unleash/evaluate` directly, update to `/api/_unleash/flags`. The response shape is also different — it now returns `{ toggles, lastUpdated, ready }`.

### Remove `unleash-client` dependency

If you had `unleash-client` as a direct dependency, remove it:

```bash
pnpm remove unleash-client
```
::
